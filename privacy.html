Privacy Policy

App: Phone Monitor  ·  Last updated: 22 March 2026

Important: Phone Monitor is a device activity logging tool. It is designed to collect detailed information about how a device is used. Before installing this app on any device, ensure that the person using that device is aware it is installed and understands what is being recorded. Installing monitoring software on a device without the knowledge of the person using it may be illegal in your jurisdiction.
Short version: Phone Monitor records extensive device activity and stores it locally. If you configure a server URL, that data is uploaded to the server you specify. Once data leaves the device, the app developer has no access to it, no control over it, and no ability to delete it. You should assume that anything uploaded to a server is permanently outside your control.

1. What Phone Monitor does

Phone Monitor is a device activity logger intended for parental monitoring or personal activity tracking. It runs as a persistent background service and records events about how the device is used. All recorded data is stored in a local database on the device. Optionally, data can be uploaded to a server of your choosing for remote review.

The developer of this app does not operate any server that receives your data. The server is configured entirely by you, and may be a server you control yourself or one operated by a third party.

2. Data collected and stored on the device

The following data is recorded continuously while the monitoring service is running:

  • App usage — the package name and display name of every app that enters or leaves the foreground, with timestamps and duration.
  • Screen events — when the screen turns on, turns off, or is unlocked, with timestamps.
  • Notifications — when a notification is posted or dismissed, including the package name of the app that sent it and whether the notification is ongoing. Notification title and message text are not recorded.
  • Phone call state — when the phone transitions between idle, ringing, and in-call states, with timestamps. Phone numbers (caller ID or dialled numbers) are not recorded.
  • SMS receipt — when an SMS message is received (requires READ_SMS permission). Message content and sender are not recorded.
  • Battery — battery level, charging state changes (plugged in / unplugged / low battery), and where available, current draw (µA) and charge capacity (µAh).
  • App installs, updates, and uninstalls — the package name and display name of affected apps.
  • Permission changes — when Android permissions are granted or revoked.
  • Usage statistics snapshots — a periodic summary (approximately every 5 minutes) of total foreground time per app since the previous snapshot.
  • Monitoring lifecycle events — when the monitoring service starts, stops, or is stopped by the user.
  • Motion and inactivity (optional, off by default) — whether the device is in motion or stationary, and periods of extended inactivity, using the accelerometer.

3. Data uploaded to a server

If you configure a server URL in the app's Settings, all of the data listed in Section 2 will be uploaded to that server in batches. Uploads are triggered by a configurable schedule (time interval, event count, or when the screen turns off).

Each upload also includes a device identifier — a cryptographic fingerprint of a key generated in the device's secure hardware — used to authenticate the upload. This identifier is unique to the app installation.

Once data is uploaded, it is beyond the app developer's control. The developer has no visibility into, access to, or ability to delete data held on any server. You should treat any data that leaves the device as permanently accessible to whoever operates that server. If the server is operated by a third party, their own privacy policy governs how they use your data. You are solely responsible for choosing a server you trust.

Data is transmitted over HTTPS and signed with a device-specific cryptographic key to protect integrity in transit. However, once received by the server, the app provides no further guarantees about how that data is stored, accessed, or shared.

If no server URL is configured, no data ever leaves the device.

4. Data we do not collect

The following data is explicitly not recorded by this app:

  • Notification title or message text
  • SMS message content or sender
  • Phone numbers (caller ID or dialled numbers)
  • Location or GPS data
  • Camera or microphone input
  • Contacts, photos, or files
  • Passwords, clipboard content, or any app screen content
  • Web browsing history

The developer does not operate any analytics service, crash reporter, or advertising network within this app. No data is sent to the developer.

5. Android permissions explained

  • READ_CALL_LOG / READ_PHONE_STATE — used to detect call state changes (ringing, in-call, idle). No call log entries or phone numbers are read.
  • READ_SMS — used to detect when an SMS arrives. Message content is not read.
  • POST_NOTIFICATIONS — required on Android 13+ to display the persistent status notification while the service is running.
  • PACKAGE_USAGE_STATS — required to identify which app is in the foreground. Must be granted manually in Android Settings.
  • BIND_NOTIFICATION_LISTENER_SERVICE — required to observe notification arrivals and dismissals. Must be granted manually in Android Settings. Title and content are not read.
  • FOREGROUND_SERVICE — required to keep the monitoring service running reliably in the background.
  • RECEIVE_BOOT_COMPLETED — allows monitoring to resume automatically after the device restarts, if it was active before.
  • INTERNET / ACCESS_NETWORK_STATE — required to upload data to a configured server.
  • REQUEST_IGNORE_BATTERY_OPTIMIZATIONS — allows the app to request exemption from battery optimisation, improving reliability of the background service.

6. Data retention and deletion

Data stored locally on the device can be deleted at any time by uninstalling the app, which removes the local database entirely.

Data that has been uploaded to a server cannot be deleted by this app. You must contact the operator of that server directly to request deletion.

7. Consent and lawful use

This app is intended for use as a parental monitoring tool (with the awareness of the monitored person where required by law) or as a personal activity logger for your own device.

You are solely responsible for ensuring your use of this app complies with applicable laws in your jurisdiction, including any laws relating to consent, workplace monitoring, and electronic surveillance. Do not install this app on a device belonging to another person without their knowledge.

8. Children's privacy

This app may be used by parents to monitor a minor child's device. In such cases, the parent or guardian is responsible for the data collected and for any server it is sent to. The app developer does not knowingly collect data about minors.
